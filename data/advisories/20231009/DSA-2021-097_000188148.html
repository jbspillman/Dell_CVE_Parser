Article Number: 000188148

DSA-2021-097: Dell PowerScale OneFS Security Update for Multiple Proprietary Component Vulnerabilities.

Summary: Dell PowerScale OneFS remediation is available for multiple security vulnerabilities that may be exploited by malicious users to compromise the affected system.

Article Content

Impact

Critical

Details
Proprietary Code CVEs Description CVSS Base Score CVSS Vector String
CVE-2021-21565 Dell PowerScale OneFS versions 9.1.0.3 and earlier contain a denial of service vulnerability. SmartConnect had an error condition that may be triggered to loop, using CPU and potentially preventing other SmartConnect DNS responses. 5.3 CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:L
CVE-2021-21553 Dell PowerScale OneFS versions 8.1.0-9.1.0 contain an Incorrect User Management vulnerability. Under some specific conditions, this may allow the CompAdmin user to elevate privileges and break out of Compliance mode. This is a critical vulnerability and Dell recommends upgrading at the earliest. 7.3 CVSS:3.1/AV:L/AC:L/PR:L/UI:R/S:U/C:H/I:H/A:H
CVE-2021-21562 Dell PowerScale OneFS contains an untrusted search path vulnerability. This vulnerability may potentially allow a user with (ISI_PRIV_LOGIN_SSH or ISI_PRIV_LOGIN_CONSOLE) and (ISI_PRIV_SYS_UPGRADE or ISI_PRIV_AUDIT) to provide an untrusted path. This untrusted path can lead to run resources that are not under the application’s direct control. This vulnerability may also impacts clusters in Compliance mode. Dell recommends upgrading at the earliest opportunity. 4.4 CVSS:3.1/AV:L/AC:L/PR:H/UI:N/S:U/C:N/I:H/A:N
CVE-2021-21563 Dell PowerScale OneFS versions 8.1.2-9.1.0.x contain an Improper Check for Unusual or Exceptional Conditions in its auditing component. This may potentially allow an authenticated user with low-privileges to trigger a denial of service event. 6.5 CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:N/I:N/A:H

Workarounds and Mitigations

CVEs Addressed Workarounds or mitigations
CVE-2021-21565 Disable or do not use SmartConnect.
CVE-2021-21553 none
CVE-2021-21562 Setting the secure_path configuration directive in the /etc/mcp/override/sudoers file replaces the user supplied PATH environment variable with the secure_path specified in the sudoers configuration file.
  1. Run the command visudo.
  2. Add the line.
Defaults secure_path="/sbin:/bin:/usr/sbin:/usr/bin:/usr/local/sbin:/usr/local/bin"
  1. Save and exit.
CVE-2021-21563 Disabling audit keeps silent failure of audits from occurring.
